# moego:writing-system-documents

## 这是什么

编写**常驻在 Agent 上下文中的系统级文档**的写作指南。

典型场景：CLAUDE.md、角色定义、项目规则、行为规范、System Prompt 中的持久指令。

## 为什么需要单独的写作方法

这类文档有一个独特属性：**compact 免疫**。它们在整个会话生命周期中驻留在上下文窗口内，每一步推理都参与 Attention 计算。

这意味着每一个 token 都是固定开销——写得越臃肿，每次推理的成本越高。所以这套原则围绕一个核心问题展开：

> **这行内容值得它的 token 成本吗？**

## 原则概览

| 编号 | 原则 | 一句话 |
|------|------|--------|
| P0 | Token 成本论证 | 每行必须通过三项驻留测试 |
| P1 | 邻近性 | 条件 + 动作 + 约束放在同一块 |
| P2 | 具体性 | 用"动词 + 宾语"代替形容词 |
| P3 | 护栏 | 先定义禁区和前置步骤 |
| P4 | 密度 | 用最少 token 传达最多信号 |
| P5 | 对齐训练分布 | 标准 Markdown、反引号、祈使句 |
| P6 | 可验证性 | 每条规则可 Yes/No 判定 |

## 使用方式

在 Claude Code 中调用：

```
/moego:writing-system-documents
```

或在编写/审查 CLAUDE.md、System Prompt 等文档时由 Agent 自动触发。
