# SPEC 增量更新策略

当模块发生变更需要更新已有 SPEC 时：

| 变更类型          | 更新动作                                                           | 是否需要重新跑 Verify |
| ----------------- | ------------------------------------------------------------------ | --------------------- |
| 新增公共 API      | 在对应场景章节补充；如涉及新的设计决策，在"关键设计约束"中新增条目 | 建议跑                |
| 新增子模块        | 场景章节补充描述；"关键设计约束"补充架构和职责划分                 | 必须跑                |
| 行为变更          | 定位受影响的约束条，修改描述；检查场景章节是否需要同步更新         | 建议跑                |
| 废弃 API / 子模块 | 从 SPEC 中彻底删除所有相关描述                                     | 不需要                |
| 新增状态/模式组合 | 在"关键设计约束"中补充规则；重新评估复杂度分级                     | 建议跑                |

增量更新后，执行 Step 8 Delivery Checklist 中的全部检查项。
