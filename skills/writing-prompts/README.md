# moego:writing-prompts

## 这是什么

编写**一次性发给 LLM 的输入**的写作指南。

典型场景：任务指令、代码请求、分析需求、问题提问、创意简报。

## 为什么需要单独的写作方法

与常驻系统文档不同，这类输入**用完即走**，不存在长期驻留的 token 成本问题。它的挑战在于另一个维度：

> **如何用一次输入就命中目标输出？**

Prompt 写得模糊，模型就会猜；猜错了就需要多轮修正。这套原则通过结构化约束，最大化单次生成的命中率。

## 原则概览

| 编号 | 原则 | 一句话 |
|------|------|--------|
| P1 | 格式和约束前置 | 先说输出格式，再给任务内容 |
| P2 | 邻近性 | 同一决策所需的信息放在一起 |
| P3 | 具体性 | 缩窄输出空间，消除歧义 |
| P4 | 示例优先 | 一个好例子胜过一段描述 |
| P5 | 单一职责 | 一个 prompt 只做一件事 |
| P6 | 角色按需 | 只在需要偏移默认行为时设定角色 |
| P7 | 分隔符 | 用标签区隔指令、上下文、数据 |
| P8 | 显式约束长度和范围 | 明确告知期望的长度与边界 |
| P9 | 参考材料在前 | 先给上下文，再提问题 |

## 使用方式

在 Claude Code 中调用：

```
/moego:writing-prompts
```

或在编写/优化 prompt、prompt 输出不符合预期时由 Agent 自动触发。
